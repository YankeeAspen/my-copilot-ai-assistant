# Security Gate

- [ ] Aucune credential/API key/token dans le repo
- [ ] Secrets charges depuis configuration securisee
- [ ] Endpoints proteges (AuthN/AuthZ) selon le besoin
- [ ] Validation input server-side en place
- [ ] Erreurs ne divulguent pas d'information sensible
- [ ] Journaux sans donnees personnelles sensibles
- [ ] Dependencies critique/haute corrigees ou mitigees
- [ ] SBOM genere (CycloneDX/SPDX) et conserve selon la politique projet
- [ ] Licences des dependances conformes (pas de licence interdite sans exception documentee)
- [ ] Rapport `security-report.md` present et traite
